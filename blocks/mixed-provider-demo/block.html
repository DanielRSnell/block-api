Provider Fallback Demo

This page demonstrates how GenerateBlocks and Gutenberg providers work together seamlessly.

What This Demo Shows

This template contains elements that will be handled by different providers:

  • GenerateBlocks providers:divs, headings, paragraphs, buttons, images
  • Gutenberg fallbacks:quotes, tables, lists, code blocks, details
  • No HTML fallbacks:Everything gets semantic conversion

The fallback system ensures you get the best of both worlds - GenerateBlocks styling where available, and comprehensive Gutenberg support for everything else.

Code Example

// This code block will use Gutenberg's CodeProvider const converter = new HtmlToBlocksConverter('generate'); const result = await converter.convertHtmlToBlocks(html); // GenerateBlocks providers take precedence // Gutenberg providers fill the gaps console.log('Conversion complete:', result.success);

Quick Info

The priority system ensures optimal block selection.

Technical Details

The fallback mechanism works by:

  1. Trying GenerateBlocks providers first (higher priority)
  2. Falling back to Gutenberg providers for unsupported elements
  3. Using HTML fallback only as last resort
Benefits
  • Semantic block conversion
  • Better editing experience
  • Consistent styling
  • No content loss

Summary

This demo shows how the block converter intelligently chooses the best provider for each element, ensuring you get semantic, editable blocks instead of generic HTML fallbacks.
